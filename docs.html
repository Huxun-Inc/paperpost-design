Design Guidelines

为研究者打造的高质量论文信息流

PaperDaily 面向每天需要追踪前沿论文、会议动态与研究趋势的用户，将复杂的学术信息组织成清晰、可信、可持续浏览的信息体验。精准推荐、实干 AI、舒服交互，让前沿研究像一款精心策划的知识杂志一样自然、克制、好读。

Philosophy

设计哲学：三个核心

PaperDaily 面向每天需要追踪前沿论文、会议动态与研究趋势的用户，将复杂的学术信息组织成清晰、可信、可持续浏览的信息体验。

精准推荐

让用户不必在海量论文中反复搜索，而是每天直接看到与自己研究方向、兴趣领域和阅读习惯相关的高价值内容。

实干 AI

AI 不停留在炫技层面，而是服务于真实研究场景：提炼论文重点、解释方法贡献、生成速读摘要，辅助用户判断是否值得深入阅读。

舒服交互

界面保持克制、清晰、顺滑。浏览、阅读、收藏、继续阅读和方向追踪一气呵成，让研究信息像日常内容消费一样自然进入工作流。

设计哲学

圆角规范

圆角是 PaperDaily Design 亲和力的核心来源。用柔软的曲线对冲学术内容的严肃感，让研究信息的消费体验更轻盈、更友好。

  • 亲和力优先：大圆角是 PaperDaily 的视觉基因，用柔软的曲线建立亲和、灵动、不尖锐的产品性格。
  • 层级对应：元素越突出、越可交互，圆角越大；容器类元素圆角适中，形成清晰的视觉层次。

Typography

低干扰排版：清楚、留白准确、层级少而稳定。大字号用于页面标题和英雄区，正文保持舒适的 15px 与 1.6 行高。

  • 层级克制：只有 Display / Heading / Body / Caption 四个层级，避免字号过多造成视觉噪音。
  • 字重分明：标题用 700-800 粗体，正文用 400 常规，辅助文字用 500 中等，对比清晰但不刺眼。
  • 行高合理：大标题行高紧凑（1.15），正文行高宽松（1.60），保证长文阅读舒适度。

Theme Colors

底色保持白色与浅灰，极淡报纸黄只作为高亮信号，冰海青和深海青负责品牌识别。整体克制，不滥用饱和色。

  • 底色中性：页面主背景是纸白与淡灰，不靠彩色撑气氛，内容本身是主角。
  • 品牌克制：深海青与冰海青只用于品牌触点和关键行动，不铺满大区域。
  • 语义清晰：建议用苔绿、提醒用赭黄、风险用校对红，各司其职，不混用。

设计哲学

无障碍阅读

PaperDaily 认为无障碍不是附加功能，而是产品的基础能力。它包含两个维度：i18n（国际化，让不同语言和文字系统的用户都能平等使用）与 a11y（可访问性，让不同能力和使用场景的用户都能顺畅阅读）。

i18n · 国际化

  • 多语言覆盖：优先覆盖全球使用人口较多的语言，同时支持重要地区语言与不同文字系统。
  • RTL 完整适配：阿拉伯语、乌尔都语等 RTL 语言从页面方向、导航位置到信息流动线全部正确镜像，混排内容不乱。
  • 文字系统适配：中日泰等无空格语言的换行与截断、越南语泰语等音调语言的行高与清晰度单独优化。
English 简体中文 繁體中文 العربية Español Français 日本語 한국어 ไทย Tiếng Việt हिन्दी বাংলা Português Indonesia اردو ……

a11y · 可访问性

  • 字号与对比度：全局字号可缩放，正文对比度满足 WCAG AA 标准，重要操作满足 AAA，色盲友好配色。
  • 键盘与读屏：完整键盘导航，焦点环清晰可见，语义化 HTML 结构，ARIA 标签准确，屏幕阅读器流畅朗读。
  • 动效可控：尊重系统"减少动效"偏好，装饰性动画可关闭，关键反馈信息同时提供文字和视觉两种通道。

  • 推荐理由放在来源行，不单独抢视觉。
  • 收藏、已读、更多类似都应就地反馈。

Components

Text Toolbar

文本排版工具要像编辑器控件，不像营销按钮；使用熟悉图标，鼠标悬停再解释含义。

  • 排版动作使用熟悉图标，鼠标悬停再解释含义。
  • 图标间距均匀，分组清晰，避免误触。
  • 激活态用轻底色反馈，不用强色块压住工具栏。

Components

Buttons & Callouts

重要按钮保持实体感，次级动作降低对比度；提示标签用轻玻璃底和细边框，避免整块高饱和色压住正文。

  • 重要按钮保持实体感，次级动作降低对比度，避免一排按钮全在抢注意力。
  • 按钮图标与文字间距一致，点击区域充足。
  • 引用、Info、Warning、Tip 用同一套玻璃态标签，只靠左侧细线和图标区分语义。

组件

Command Bar

桌面端不要只给搜索框，要给命令入口，支持跳页、查论文、开 AI 分析。

  • 搜索、跳转、动作合并在一个入口里。
  • 空输入也要给推荐动作，不要空白。
  • 支持键盘提示，降低桌面端操作成本。

Components

Breadcrumb

面包屑导航提示用户当前位置，层级清晰、轻量不抢注意力。

  • 导航提示要轻，不要让辅助控件抢正文的戏。
  • 分隔符用细箭头，不用粗竖线。
  • 当前页用粗体，不做链接。

Components

Tooltip

轻量提示气泡，鼠标悬停或聚焦时出现，解释图标或缩略信息。

  • 只放短句解释，不放大段内容。
  • 气泡用轻玻璃底，带细边框。
  • 暗黑模式下保持可读，对比度足够。
展示引用来源与模型推断的区别。

Components

Menu

下拉菜单承载更多动作，图标+文字清晰分组，危险动作单独标识。

  • 菜单项左对齐图标和文字，间距一致。
  • 危险动作用强调色，放在菜单底部。
  • 悬停态用轻底色，不用强色块。

Components

Dialog

模态对话框用于确认重要操作，标题、说明、动作按钮层级清晰。

  • 弹层内部维持白底或深底的清晰层级。
  • 主动作在右，次动作在左，按钮顺序一致。
  • 移动端和暗黑模式下均可读。

Toast

胶囊形筛选标签，用于快速切换分类、主题或视图。

  • 图标+文字，语义一目了然。

Search Input

组件

Bottom Sheet

移动端反馈、筛选和更多操作走底部面板，别把卡片长按菜单做得太隐蔽。

  • 主动作放上半区，危险动作放底部。
  • 能单手点到，不挤压正文阅读。
  • 关闭动效要干净，不要拖泥带水。

组件

AI Answer Block

AI 回复不是一大坨聊天文本，而是回答、证据和动作三层结构。

  • 主答案先短后长，默认给判断。
  • 引用片段和模型推断必须分层显示。
  • 回答后面直接给继续分析动作。
