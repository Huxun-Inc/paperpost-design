# ADR 0001: 产品体验方向

## 状态

Accepted

## 背景

传统论文工具普遍强调检索、管理和严肃学术感，但用户的真实日常需求往往是：

- 快速知道今天哪些论文值得看。
- 用低成本理解论文核心价值。
- 把值得读的内容沉淀下来。
- 让系统越来越懂自己的研究兴趣。

因此，论文邮报不应只做“论文列表 + 摘要 + 收藏”，而应成为一个研究信息消费和兴趣推荐产品。

## 决策

采用“高级推荐信息流 + 编辑化阅读”的体验方向。

- 首页以推荐 feed 为核心。
- 详情页以 plain language 理解为核心。
- 收藏、听读、更多类似、不感兴趣都是推荐系统反馈。
- 数据面板服务用户掌控感，而不是后台统计展示。
- 四端共享内容模型，交互按平台优化。

## 结果

正向影响：

- 降低用户打开和阅读成本。
- 更容易形成每日使用习惯。
- 推荐反馈链路更完整。
- 视觉上区别于传统学术工具。

风险：

- 过度信息流可能让产品显得不够严谨。
- 推荐系统如果解释不足，用户会失去信任。
- 摘要生成质量决定核心体验。

应对：

- 保留来源、作者、原文、时间和可信度标记。
- 每条推荐尽量提供短理由。
- 摘要分层展示，并提供纠错入口。
