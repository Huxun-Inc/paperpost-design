# 交互原则

## 推荐体验

推荐不是单独的算法模块，它应该渗透在前端的每个反馈点。

### 明确信号

- 收藏：强正反馈，代表长期价值。
- 点开详情：中等正反馈，代表标题或摘要有效。
- 读完 / 停留：强理解信号，但要结合内容长度归一化。
- 跳过：弱负反馈，不能立刻判定不喜欢。
- 不感兴趣：强负反馈，需要允许用户选择原因。
- 更多类似：明确的兴趣扩展。
- 关注关键词 / 作者 / 领域：长期偏好信号。

### 隐式节奏

- 信息流默认不要求用户做选择，先给内容。
- 每次显式反馈后，应该在 1 到 3 篇内体现变化。
- 新领域探索要用少量插入，不要突然重置用户的 feed。
- 推荐理由要短，比如“因为你收藏了 scaling law”和“来自你关注的 arXiv AI”。

## 留存路径

### 日常打开

1. 看今天最值得读的 5 到 12 篇。
2. 快速收藏或跳过。
3. 点进 1 到 2 篇看 plain language 摘要。
4. 通勤或碎片时间听读。
5. 晚上回到收藏夹或专题继续深读。

### 关键回访钩子

- 今日 briefing：固定时间的轻量新鲜感。
- 继续阅读：回到上次没读完的论文或专题。
- 趋势提醒：某个关键词突然变热。
- 作者动态：关注作者有新论文。
- 周总结：本周你追踪了哪些主题，错过了哪些高价值论文。

## 信息流手感

- 卡片之间要有明确节奏，不能全都一样重。
- 第一屏至少露出 2.5 张内容卡，让用户有继续滑动的冲动。
- 每张卡只承载一个主判断：要不要读。
- 摘要最多两到三行，详情页再展开解释。
- 标签是兴趣入口，不是数据库字段展示。

## 详情页手感

详情页回答四个问题：

- 这篇讲什么？
- 关键发现是什么？
- 为什么重要？
- 下一步我可以做什么？

推荐结构：

- 标题和来源。
- Key Finding 高亮卡。
- In Plain English 列表。
- Why It Matters。
- Methods / Caveats 可折叠。
- 原文、收藏、下一篇、听读。

## 微交互

- 收藏：轻微弹性和星标/书签填充，立即可撤销。
- 下一篇：横向滑入，保持阅读节奏。
- 反馈菜单：底部 sheet，避免打断。
- 听读：迷你播放器常驻底部，详情页可展开。
- 刷新：不要只转圈，展示“正在找今天值得读的论文”这类状态，但文案要短。

## 空状态

空状态不能像错误页。它应该给出下一步：

- 没有收藏：展示“先收藏 3 篇，系统会开始理解你的研究口味”。
- 没有推荐：提供领域选择和导入关键词。
- 没有网络：保留离线收藏和最近阅读。
