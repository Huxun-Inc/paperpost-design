# 四端适配

## 总原则

同一套内容模型，不同端用不同信息密度。

- iOS / Android：沉浸、轻反馈、滑动浏览。
- Web：快速浏览、搜索、分享、轻整理。
- mac：高密度阅读、批量管理、键盘操作、多栏工作流。

## iOS

- 使用底部 Tab，主路径不超过 5 个入口。
- 详情页支持右滑返回和横滑下一篇。
- 收藏、更多类似、听读应在拇指热区。
- 动效轻而明确，避免长时间遮挡内容。
- 支持系统分享、动态字体和离线收藏。

## Android

- 尊重返回键和 Material 手势。
- 底部导航可保留，但反馈 sheet 要符合 Android 预期。
- 长按卡片打开反馈菜单。
- 通知更克制，优先日报和关注主题提醒。

## Web

- 响应式断点：
- `mobile`: 360-767。
- `tablet`: 768-1023。
- `desktop`: 1024-1439。
- `wide`: 1440+。

布局：

- 窄屏同移动端。
- 中屏 feed + detail preview。
- 宽屏三栏：导航 / feed / reader。

Web 特性：

- URL 可分享到具体论文。
- 支持命令菜单。
- 搜索和筛选更强。
- hover 露出快捷操作，但不依赖 hover 完成核心任务。

## mac

mac 端应该像研究者每天打开的“论文工作台”，不是放大的手机。

布局：

- 左侧 source list：日报、领域、作者、收藏、稍后读。
- 中间 feed：高密度卡片列表。
- 右侧 reader：当前论文摘要和原文入口。
- 底部或右下：听读队列。

交互：

- 键盘导航是一级能力。
- 支持多选、批量收藏、批量标记已读。
- 支持分屏和窗口尺寸变化。
- 支持拖拽论文到收藏夹或专题。

## 跨端一致性

保持一致：

- 内容字段。
- 视觉 token。
- 收藏和推荐反馈语义。
- 关键页面顺序。
- 论文详情的信息结构。

允许不同：

- 导航形式。
- 屏幕密度。
- 快捷操作。
- 动效幅度。
- 多任务能力。
