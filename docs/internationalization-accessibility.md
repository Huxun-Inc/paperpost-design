# 暗黑模式、多语言与 RTL 规范

## 设计目标

暗黑模式、多语言和 RTL 不是后期皮肤，而是基础布局能力。论文日报要支持用户在不同语言、不同阅读方向和不同光照环境下保持同样舒服的阅读节奏。

## 暗黑模式

原则：

- 不做纯黑背景，使用接近墨黑的深灰，降低长时间阅读疲劳。
- 卡片层级通过边框、轻阴影和明度差表达，不依赖高饱和蓝色。
- 蓝色在暗黑模式中提高明度，避免低对比。
- 论文正文区域优先保证对比度，不追求炫酷。

必须覆盖：

- 页面背景。
- 卡片、输入框、底部导航。
- 关键发现高亮卡。
- 标签 chip。
- 收藏/更多类似等反馈按钮。
- 图表和数据面板。

## 多语言

内容分层要支持不同文本长度：

- 中文通常较短，英文标题较长，阿拉伯语可能需要更高行高。
- 卡片标题最多显示 2 到 3 行，超出进入详情。
- 按钮文案要预留 30% 到 50% 扩展宽度。
- 不把固定像素高度写死在长文本容器上。
- 使用语义键值管理文案，不在组件内硬编码。

推荐语言键：

- `home.dailyBriefing`
- `paper.recommendationReason`
- `paper.keyFinding`
- `paper.plainEnglish`
- `feedback.notInterested`
- `feedback.moreLikeThis`
- `auth.continueWithApple`
- `onboarding.importKeywords`

## 阿拉伯语与 RTL

阿拉伯语界面必须设置：

```html
<html lang="ar" dir="rtl">
```

布局规则：

- 导航从右到左排列。
- 卡片主要内容从右侧开始。
- 收藏按钮移动到卡片左侧边缘。
- 返回箭头、下一篇箭头等方向性图标需要镜像。
- 非方向性图标，如星标、刷新、耳机，不需要镜像。
- 数字、英文论文名、arXiv 分类在阿语文本中要保持可读。

CSS 建议：

```css
[dir="rtl"] .paper-card {
  padding-inline: 16px 44px;
}

[dir="rtl"] .save-button {
  inset-inline-start: 12px;
  inset-inline-end: auto;
}
```

## 混排

论文产品天然包含多语言混排：

- 英文标题。
- 中文或阿拉伯语解释。
- arXiv 分类。
- 数字、日期、公式。
- 作者名和机构名。

规范：

- 不强行翻译论文标题，除非用户明确选择翻译。
- 来源、作者、DOI、arXiv ID 保持原文。
- Plain English / 白话解释 / Arabic summary 可以本地化。
- 标签可以同时保留英文原始术语和本地化解释。

## 可访问性

基础要求：

- 所有交互控件可键盘聚焦。
- 收藏、播放、切换主题等图标按钮必须有可访问名称。
- 文本对比度达到 WCAG AA。
- 动效尊重 `prefers-reduced-motion`。
- 语言切换要更新 `lang` 和 `dir`。
- 表单错误提示需要关联到输入框。

## 检查清单

- 中文、英文、阿拉伯语都不会撑破卡片。
- RTL 下收藏按钮、导航、箭头方向正确。
- 暗黑模式下蓝色标签和正文对比足够。
- 长英文标题不会覆盖摘要。
- 表单、chip、反馈按钮在所有语言下都可点击且不换行崩坏。
