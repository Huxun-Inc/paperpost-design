# 组件系统

## 基础原则

- 卡片用于重复内容项，页面区域不要堆叠成卡片套卡片。
- 操作按钮尽量图标优先，关键 CTA 才使用文字按钮。
- 组件状态必须支持推荐反馈：默认、已读、已收藏、不感兴趣、推荐解释。
- 移动端重手势，桌面端重快捷操作和多栏效率。

## App Shell

### Mobile

- 顶部：日期 / 标题 / 刷新。
- 主体：论文卡片信息流或详情。
- 底部：Home、Data、Saved、Listen、Settings。
- 浮层：反馈、筛选、听读队列。

### Desktop / Web

- 左侧：频道、关键词、作者、收藏夹。
- 中间：feed 或阅读列表。
- 右侧：摘要、推荐理由、相关论文、趋势。
- 顶部：搜索、同步、账户和模式切换。

## Paper Card

用途：帮助用户快速决定“读、不读、收藏、看更多类似”。

字段：

- 排名或推荐强度。
- 来源领域，如 `ArXiv: AI`、`Nature`。
- 标题。
- 一句话 plain language 摘要。
- 作者或机构，可在长按/hover 时展开。
- 标签。
- 收藏图标。
- 推荐理由，可选。

状态：

- `default`：普通推荐。
- `hot`：今日重点，允许更强标题和蓝色编号。
- `read`：降低对比度但保留入口。
- `saved`：书签/星标高亮。
- `muted`：用户选择不感兴趣后收起。

## Key Finding Card

用途：详情页第一块理解锚点。

规范：

- 使用浅蓝或浅灰底，不使用大面积高饱和色。
- 标题短，正文 2 到 4 行。
- 可以带一个灯泡、靶心或指南针类图标。
- 保持可信口吻，不做夸张营销文案。

## Plain English List

用途：把论文转成可扫读结论。

规范：

- 每条用蓝色小圆点或细线起始。
- 加粗每条的结论短语，后接解释。
- 一屏内优先 3 到 5 条。
- 长解释进入展开态。

## Feedback Controls

必须支持：

- 收藏。
- 不感兴趣。
- 更多类似。
- 关注关键词。
- 标记已读。
- 分享。

移动端：

- 主反馈用手势和底部 sheet。
- 长按卡片打开反馈菜单。

桌面端：

- hover 时露出快捷反馈。
- 支持键盘：`S` 收藏，`J/K` 上下篇，`M` 更多类似，`X` 不感兴趣。

## Data Dashboard

用途：给用户掌控感，而不是做后台报表。

模块：

- 今日论文数。
- 预计阅读时间。
- 关键词热度。
- 作者活跃度。
- 本周兴趣变化。
- 已收藏未读。

视觉：

- 数字要大，但每屏只强调 1 到 2 个关键数字。
- 图表轻量，优先条形、趋势线、排行榜。
- 不做复杂 BI 风格。

## Listen Player

用途：把论文从阅读任务变成陪伴式消费。

组件：

- 迷你播放器。
- 当前论文标题。
- 章节进度：摘要、关键发现、plain English、方法、局限。
- 倍速。
- 下一篇。

体验：

- 从列表和详情都能一键听。
- 切出 App 后保持上下文。
- 听完后给出收藏或更多类似。
