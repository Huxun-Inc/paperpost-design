# 视觉语言

## 品牌关键词

- Editorial
- Fresh
- Intelligent
- Calm
- Addictive in a healthy way

## 色彩

主色不是“学术蓝铺满屏幕”，而是纸白、墨黑、蓝色知识锚点和少量状态色。

- 背景：接近纸张的白，不偏黄。
- 正文：高对比墨黑。
- 次级文本：中性灰。
- 强调蓝：用于可点击、推荐锚点、收藏状态、编号。
- 成功绿：只用于同步、完成、正向状态。
- 警示橙：只用于提醒和异常，不用于品牌。

## 字体

字体方向要避开“论文期刊”和“学校数据库”的感受。不要使用 Times New Roman、新罗马、传统宋体式大标题，也不要为了显得专业而让界面变得疏离。

- 品牌标题：Noto Sans SC Black / Bold，或同类现代、饱满、干净的无衬线。
- 英文标题：使用现代 sans，不使用传统学术 serif。
- 中文标题：Noto Sans SC Bold / Black。
- 正文和 UI：SF Pro、Inter、Noto Sans SC。
- 数字：使用 UI 字体，保证 dashboard 清楚。

原则：

- 大标题可以有内容 App 的冲击力，但不要有期刊封面的距离感。
- 列表正文必须高可读。
- 按钮和导航不使用 serif。
- 不使用负字距。

## 卡片

- 圆角控制在 8px 左右。
- 阴影非常轻，主要靠边框和层级。
- 列表卡片高度稳定，避免每张卡高度差过大。
- 卡片内只放一个主要判断任务。

## 图标

图标应承担功能，不做装饰堆砌。

常用：

- Home
- FileText
- Star / Bookmark
- Headphones
- Settings
- Refresh
- Search
- More
- ChevronRight
- Sparkles 用于推荐解释，但要少用。

## 动效

动效原则：

- 服务节奏，不炫技。
- 收藏、下一篇、刷新、sheet 展开必须有反馈。
- 信息流滚动保持稳定，不做大幅视差。
- 详情切换可以轻微横向过渡。

推荐时长：

- 点击反馈：120-180ms。
- 页面进入：180-260ms。
- 底部 sheet：220-320ms。
- 刷新状态：不低于 400ms，避免闪烁。
